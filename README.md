# Post-Compromise Attack Simulation (Educational)

## ⚠️ Disclaimer
This project is an educational simulation of post-compromise attacker behavior using native Windows tools.  
It is intended strictly for cybersecurity awareness, training, and research purposes inside controlled environments such as virtual machines.  

No real malware, exploits, or destructive actions are included.

---

## 🎯 Project Overview
This simulation demonstrates how attackers may operate **after gaining initial access** to a system.

The script showcases a multi-stage attack workflow using only built-in Windows capabilities.

---

## 🔥 Demonstrated Attack Stages

- Defense Evasion (disabling protections)
- Credential & Data Discovery
- Data Staging
- Exfiltration Window Simulation
- Dwell Time (Delayed Action)
- Payload Execution
- Anti-Forensics (Trace Removal)
- System Restoration

---

## 🧠 Learning Objectives

- Understand post-compromise attacker behavior
- Recognize risks of plaintext credential storage
- Observe how native tools can be misused
- Learn why monitoring after intrusion is critical

---

## 🖥️ Requirements

- Windows system
- Administrator privileges
- PowerShell
- Recommended: Isolated Virtual Machine

## 🔒 Safety Notes
